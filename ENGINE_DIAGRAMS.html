3. Sequence — recommendation request

sequenceDiagram
  participant C as Client
  participant N as Nginx
  participant A as FastAPI
  participant P as PostgreSQL
  participant G as GraphClient
  participant R as Redis
  participant J as Neo4j
  participant K as Kafka
  C->>N: POST /v1/recommendations
  N->>A: proxy + rate limit
  A->>A: API key check
  A->>P: get_for_scoring
  P-->>A: PatientProfile
  A->>A: Pipeline evaluate
  loop Cacheable KG queries
    A->>G: nutrient / baseline / guideline
    G->>R: GET cache
    alt hit
      R-->>G: cached
    else miss
      G->>J: Cypher
      J-->>G: result
      G->>R: SETEX
    end
  end
  loop Safety queries
    G->>J: interactions
  end
  A->>P: save session audit snapshot
  opt KAFKA enabled
    A->>K: recommendation.served
  end
  A-->>C: 200 recommendations
